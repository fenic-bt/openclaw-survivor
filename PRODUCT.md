# Product Vision: OpenClaw Survivor Pro

## 核心定位

> **"Don't let your AI agent die. We bring it back."**

目标用户：技术水平 ≤ 云瑞的普通用户，他们的 AI Agent 养死了只能干瞪眼。

## 产品分级

### Free（开源版）
- `snapshot.sh`（已开源）
- 手动快照
- 命令行恢复
- 任何 OpenClaw 用户免费使用

### Pro（付费版 $9/月 或 $79/年）
核心差异：**自动濒死检测 + 一键恢复**

| 功能 | Free | Pro |
|------|------|-----|
| 手动快照 | ✅ | ✅ |
| 自动每日快照 | ✅ | ✅ |
| **濒死自动检测** | ❌ | ✅ |
| **一键恢复（小白界面）** | ❌ | ✅ |
| **云端同步** | ❌ | ✅ |
| **跨设备恢复** | ❌ | ✅ |
| 快照数量 | 30个 | 无限 |
| 恢复历史 | 30天 | 1年 |

## 濒死检测逻辑

监控以下信号：
1. OpenClaw 连续 N 次启动失败
2. `~/.openclaw/workspace/MEMORY.md` 内容异常（乱码/丢失）
3. OpenClaw 配置损坏
4. 关键文件被删除

检测到异常 → 触发告警 → 提供一键恢复按钮

## 商业模式

- **获客**：OpenClaw 用户群（Discord/论坛）
- **转化**：养死过1次 + 不想再经历第2次的用户
- **定价**：$9/月 ≈ 一杯咖啡，目标用户付得起

## 竞争壁垒

1. 先发优势：已经开源了，传播快
2. 生态锁定：恢复越多次，快照积累越多，越不想换
3. 网络效应：用户多了 → 案例多了 → 更多用户信任
